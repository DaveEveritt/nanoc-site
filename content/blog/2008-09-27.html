nanoc 2.1.3 is now available. This bug-fixing release fixes a few issues I'll quickly explain.

First of all, there was a bug which would prevent Haml and Sass filters from being used in assets. Secondly, layouts named "index" are now handled correctly (due to backward compatibility, "index" was handled differently). Finally, the filesystem_combined data source now supports assets (a bug prevented assets from working there).

Additionally, the autocompiler now serves index files when requesting a directory (quite similar to Apache's DirectoryIndex or lighttpd's index-file.names). Index files that are not created by nanoc but still appear in the output directory (as the result of a rake task, for example) will therefore be served instead of returning a 404.

You can update your copy of nanoc the usual way by using the "gem update" command like this:

> sudo gem update nanoc

If you have any trouble with this nanoc release, please do let me know on the mailinglist. You can always report bugs on Trac (preferred), on Bitbucket and even on the brand new nanoc project site at Kenai. Thanks!
