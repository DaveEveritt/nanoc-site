nanoc 2.1.6 has been released. This version fixes a handful of bugs and improves compatibility with third-party utilities.

Pages and assets that use the filesystem_combined data source can now have an empty metadata section. Previously, you'd get weird nil errors and the like.

The rdoc filter now supports both RDoc 1.x and RDoc 2.x.

The autocompiler has been improved in several ways. First of all, it serves asset content correctly. Previously, nanoc would sometimes simply serve outdated compiled data instead.

Secondly, the asset compiler now reports errors more accurately. When an exception occurs somewhere in the asset compiler, it will be reported.

Lastly, the autocompiler no longer serves index files (such as index.html) if a request is made for a directory and the request path does not end with a slash. For example, the file about/index.html will not be served when the request path is /about instead of /about/.

You can, as always, update your copy of nanoc by issuing the "gem update" command, like this:

> sudo gem update nanoc

If you have any issues with this nanoc release, do let me know on the mailinglist or on the IRC channel. Enjoy!
